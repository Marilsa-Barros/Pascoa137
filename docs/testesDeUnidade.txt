Enxegar uma Piramide de Teste onde fica a camda de UI é uma visão de Front End

Usar uma massa de teste --> DDT = Data Driven Test = Teste Direcionado por Dados

TDD = Test Driven Development = Desenvolvimento Direcionado por Testes

Daria para criar os testes sem que houvesse as funções prontas

Você teria:
- nome da função que vai testar. Exemplo: somarDoisNUmeros
- os parametros da função. Exemplo: num1e num2, que são do tipo double
- retorna um doube
- a informação sobre o funcionamento Ex. Seria uma soma de dois numeros
